\section{Introduction}
\vspace{-6pt}
Model-Based Systems Engineering is central to sustainable digital engineering practice. With SysMLv2, textual modeling enables version control, automation, and integration into modern development environments. Prior practical work has already shown that generative AI can accelerate textual model authoring.

This paper investigates how ontology-driven SysMLv2 library engineering can be scaled while preserving quality and reproducibility.

\section{Approach}
\subsection{Architecture of the Model Space}
The model space is organized into Configuration Items (CIs), each representing a bounded domain and exposing controlled package dependencies. This structure supports parallel evolution while preserving architectural consistency.

\subsection{Integration of FIBO}
The Financial Industry Business Ontology (FIBO) is mapped into SysMLv2 and specialized by domain. Roles and types are aligned consistently so that business semantics remain explicit and machine-processable.

\subsection{Agentic Governance}
The modeling workflow separates methodological and syntactic accountability through distinct steward roles. This reduces the risk of semantically weak but syntactically valid model increments.

\subsection{Automation and Build Path}
Automation covers structure documentation, parser-critical cleanup, lockfile maintenance, and package publication using Sysand. The reproducible flow is based on model update, metadata validation, and package build.

\section{Example Application: FiBO2SysMLv2 Library}
The FiBO2SysMLv2 library is used as a reference artifact for ontology-centered SysMLv2 library engineering. The repository integrates multiple financial domains and demonstrates reproducible package generation.

\section{Discussion}
Ontologies are not supplementary context but the core prerequisite for actionable models. AI contributes most when embedded in explicit quality constraints, validation checks, and role clarity.

\section{Summary and Outlook}
The combination of textual SysMLv2, FIBO integration, CI structuring, and governed AI-assisted modeling forms a scalable basis for executable and verifiable enterprise model libraries.

Future work includes tighter OWL-to-SysMLv2 semantic alignment, AI-assisted consistency checks, and automated generation of verification evidence.
